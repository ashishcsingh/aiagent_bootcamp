# Lesson 6: Key Agent Concepts Glossary

This glossary compiles the core architectural paradigms, design patterns, and operational constructs of modern autonomous AI agent systems.

---

## 1. Core Loop & Execution Patterns

### Agentic Loop
The continuous, stateful cycle of perception, reasoning, planning, and action. Unlike standard stateless chatbot request-response cycles, an agent loop runs iteratively until it determines its goal has been successfully completed.

### ReAct (Reasoning & Acting)
A prompting paradigm where the model explicitly generates its reasoning (*Thought*) before executing a function (*Action*) and receiving the result (*Observation*). This cycle repeats dynamically to solve complex, multi-step tasks.

### Function / Tool Calling
The capability of a model to parse user instructions, choose one or more appropriate tools, and construct a structured JSON argument payload matching the registered tool's schema definition.

---

## 2. Planning & Self-Correction

### Self-Reflection / Self-Correction
A feedback loop where the agent evaluates the results of its own actions (such as tool execution logs or compilation errors). If a tool fails, the agent uses the error output to adjust its planning parameters and tries again rather than crashing.

### Planning Topologies
The cognitive pathways an agent uses to solve a task:
*   **Linear/Sequential**: Breaking a goal down into chronological sub-tasks.
*   **Tree of Thoughts (ToT)**: Generating and evaluating multiple alternative reasoning paths simultaneously, allowing backtracking.
*   **Graph/Cyclic**: Navigating a dynamic network of states and conditional edges.

---

## 3. Memory Architectures

### Short-Term Memory
The active in-context conversation history and intermediate ReAct steps stored directly inside the LLM's active context window.

### Long-Term Memory
Persisted vector databases, document indexes, or relational key-value logs that allow an agent to recall context, facts, and user details across different session lifetimes.

### Episodic Memory
A specialized record of past execution runs, success paths, and tool calling results that helps the agent optimize its plans based on what worked in previous similar runs.

---

## 4. Multi-Agent Systems & Topologies

### Sequential Orchestration (Pipeline)
A multi-agent flow where tasks pass linearly from one specialized agent to the next (e.g., Researcher $\rightarrow$ Writer $\rightarrow$ Editor).

### Hierarchical Orchestration (Manager-Worker)
An orchestration structure where a "Manager" agent coordinates instructions, delegates sub-tasks to specialized "Worker" agents, reviews their results, and synthesizes the final reply.

### Networked / Conversational Graphs
A fluid structure where multiple agents interact dynamically based on state transitions, shared events, and message passing (allowing cyclic loops and complex multi-agent negotiations).

### State Blackboard
A centralized database or global shared state (like in LangGraph) where all active agents in a system read context and write updates, rather than relying strictly on direct peer-to-peer messaging.

---

## 5. Security & Operations

### Model Context Protocol (MCP)
An open standard developed by Anthropic that establishes a universal JSON-RPC protocol over stdio or HTTP, allowing any MCP host client (like an IDE or agent app) to securely consume tools, resources, and prompt templates from any registered MCP server.

### Execution Sandboxing
Running untrusted code generated by agents (such as Python scripts or bash commands) in an isolated, restricted secure runtime environment (like Docker containers, WebAssembly hosts, or gVisor VMs) to protect the host system.

### Human-in-the-Loop (HITL)
An operational design pattern where sensitive actions (such as sending emails, initiating financial transactions, or executing database drops) are paused until explicitly approved or corrected by a human administrator.

### Prompt Injection
A security vulnerability where untrusted input from tools, parsed web pages, or email bodies overrides the agent's core system instructions, causing it to execute unauthorized commands or leak private context.
